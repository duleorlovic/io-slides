Working with Rails requires multiple windows

  • One terminal window for rails s
  • One for rails test or running guard
  • One for rails console or bin/rails tasks
  • One for background jobs bin/delayed_job run
  • One for productionn commands like heroku logs
  • One for code snippets where you can search and copy your earlier code
  • Finally one for your vim or any other editor
  • At the end we need one or more Chrome or Firefox windows
  • each with tabs and their own Developer Tools window
  • If you are true freelancer than you have all those windows opened for multiple client projects in the same time

Keyboard shortcuts to switch windows

No more Alt + Tab

Specific for each Operating System

  • For Linux you can use xdotool with which you can add classes to windows, search and activate specific window
  • For MAC you can use AppleScript to tell Application to activate specific window id.
  • For windows ...please create pull request...
  • I usually like to map Alt + j (or other jkl; keys) so I can jump directly from any window to any other window

Start all project windows at once

When computer boots up you should be able to open all windows in one go and each window with it's specific command. 

start_all_windows() {
  projectPath=${1-$(pwd)}
  port=300$(get_current_viewport)
  url=http://localhost:$port
  projectName=`basename $projectPath`

  s $projectPath j
  s $projectPath k 120x24+830+100 ""
  win_width=`expr $(monitor_size) / 36` # 7200 / 36 = 200
  s ~/jekyll/blog l 80x24+1250+0 # right: 80x24-0+100 # 80x24+1250+100

  start_browser h $url firefox "Mozilla Firefox"
  start_browser u $url google-chrome Google 0,400,100,-1,-1
}

Load configuration for each project

Load all environment keys which are used for that project (for example define ports and start Neo4j server) 

function keys {
  current_path=$(pwd)
  projectName=`basename $current_path`
  if [ -f ~/config/keys/$projectName.sh ];then
      source ~/config/keys/$projectName.sh
  else
    echo can not find ~/config/keys/$projectName.sh
  fi
}
Bonus hint: Run rails server on specific port rails s -p 3003 depending on which viewport we are currently, so we can open browser for that specific http://localhost:3003
